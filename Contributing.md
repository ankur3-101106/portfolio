# Contributing to This Portfolio

Thank you for your interest in contributing to this portfolio website. Contributions, suggestions, and improvements are always welcome.

## Ways to Contribute

You can contribute by:

* Reporting bugs or broken features
* Suggesting UI/UX improvements
* Improving documentation
* Optimizing performance or accessibility
* Fixing typos or code issues
* Enhancing cybersecurity-related content or demonstrations
* Adding useful features or tools

## Before You Start

Please make sure to:

1. Read the `CODE_OF_CONDUCT.md`
2. Check existing issues or discussions before creating a new one
3. Keep contributions professional and constructive

## Development Setup

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/your-portfolio.git
cd your-portfolio
```

### 2. Install Dependencies

```bash
npm install
```

### 3. Start Development Server

```bash
npm run dev
```

or if using Docusaurus:

```bash
npm run start
```

## Branch Naming

Use clear and descriptive branch names:

```bash
feature/add-contact-form
fix/navbar-overflow
docs/update-readme
```

## Commit Message Guidelines

Write meaningful commit messages:

```bash
feat: add animated hero section
fix: resolve mobile navbar issue
docs: improve setup instructions
```

## Pull Request Process

1. Fork the repository
2. Create a new branch
3. Make your changes
4. Test your changes properly
5. Commit with clear messages
6. Push your branch
7. Open a Pull Request

## Coding Standards

Please follow these guidelines:

* Write clean and readable code
* Keep components modular
* Use consistent formatting
* Avoid unnecessary dependencies
* Maintain responsive design
* Follow accessibility best practices
* Comment complex logic when necessary

## Security Policy

If you discover a security vulnerability:

* Do **not** publicly disclose it immediately
* Contact the repository owner privately
* Provide detailed reproduction steps if possible

## Contribution Rules

Contributions may be rejected if they:

* Include malicious code
* Break existing functionality
* Violate the Code of Conduct
* Introduce unethical or illegal functionality
* Contain plagiarized content

## Style & Design Notes

This portfolio focuses on:

* Minimal and modern UI
* Performance optimization
* Cybersecurity-oriented content
* Accessibility and responsiveness
* Clean developer experience

Please keep contributions aligned with the overall design philosophy.

## Need Help?

If you have questions, feel free to open an issue or discussion in the repository.
